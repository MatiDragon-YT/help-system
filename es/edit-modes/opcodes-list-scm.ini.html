## Formato del Archivo Las líneas que comienzan con `;` se ignoran. ### Metadatos Los parámetros INI especiales proporcionan información de metadatos sobre la lista de opcodes.\ \ `VERSION` – define la versión de este archivo. Vea la directiva [$VERSION](../coding/directives.md#usdversion)\ `PUBLISHER` – quién es el autor de esta versión\ `DATE` – la fecha de actualización. El desensamblador imprime los metadatos en la primera línea del archivo de salida. ### Definición del Opcode Cada línea tiene la siguiente sintaxis:\ \ `xxxx=N, yyyy` \ `XXXX` – el número de \ `N` – la cantidad de parámetros\ `yyyy` – la descripción del opcode. El número de opcode y el número de parámetros son definitivos y no se pueden cambiar. La descripción está abierta a modificaciones. Si el archivo INI contiene múltiples definiciones para el mismo opcode, este último controla. Por convención, una descripción de un opcode condicional comienza con dos espacios. ### Parámetros del Opcode Un parámetro es una parte dinámica del opcode. Empiezan y terminan con `%` ``` 0001=1,wait %1d% ms ``` Esta línea es la definición del opcode `0001`. Cuando el desensamblador encuentra `0001` en un script, imprime las palabras `wait` y `ms` con el valor del parámetro en el medio del archivo de salida. Un parámetro consta de dos partes: un 'número' y un 'tipo'. #### Orden de los Parámetros El número en el parámetro `%1d%` es su índice. La idea es que algunos opcodes tengan sus parámetros reorganizados para hacer que el código fuente sea más legible. ``` 0053=5,%5d% = create_player %1o% at %2d% %3d% %4d% ``` El desensamblador escribe el quinto parámetro del opcode antes que cualquier otro. ```sb3 0053: $PLAYER_CHAR = create_player #NULL at 2488.562 -1666.865 12.8757 ``` Si los parámetros siguen su orden original, el opcode `0053` se vería así: ```sb3 0053: create_player #NULL at 2488.562 -1666.865 12.8757 $PLAYER_CHAR ``` {% hint style="warning" %} Debe evitarse por todos los medios cambiar el orden de los parámetros en el archivo INI, ya que hace que el código fuente sea incompatible con las definiciones de opcode predeterminadas o scripts anteriores. {% endhint %} {% hint style="info" %} El orden de los parámetros puede variar de un modo a otro. Consulte la tabla de comparación en [Editar modos](README.md#available-modes) para averiguar qué modo usa qué tipo. {% endhint %} #### Tipos de Parámetros Una letra después del número de índice indica el tipo de parámetro.\ \ `d` - cualquier valor\ `p` - referencia a una [etiqueta](../coding/data-types.md#etiquetas)\ `o` - cualquier id de [modelo](../coding/data-types.md#nombres-de-modelos) (incluyendo objectos)\ `m` - el id de un modelo en `.ide` (no objectos)\ `g` - una clave del archivo [`.gxt`](README.md#text)\ `x` - un ID de script externo\ `k` - una cadena literaria de tamaño fijo con terminal nula (128 bytes) {% hint style="info" %} Puede cambiar libremente los tipos de parámetros, si es necesario.{% endhint %}
