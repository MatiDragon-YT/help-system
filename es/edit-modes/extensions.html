Una **extensión** es un conjunto de nuevas instrucciones añadidas al juego básico. La [librería CLEO](https://cleo.li) y sus complementos son la forma más conocida de agregarlos. Para que Sanny Builder conozca las instrucciones personalizadas y señale explícitamente su uso intencional, el script debe tener la [directiva](../coding/directives.md#$use) `{$USE}`. La directiva `{$USE}` habilita un conjunto particular de instrucciones (consulte a continuación la [lista de extensiones](extensions.md#extensions-list)). ``` {$USE CLEO} {$USE ini} {$USE CLEO+} ``` Cada opcode incluido en los [archivos INI] de Sanny Builder (opcodes-list-scm.ini.md) ha sido categorizado para pertenecer a alguna extensión. Actualmente se hace a través del archivo `extensions.txt`. {% hint style="info" %} Vea [#74](https://github.com/sannybuilder/dev/issues/74) para obtener más información sobre cómo se espera que funcione esta característica en el futuro. {% endhint %} ## Extensiones Implícitas Los opcodes estándar del juego se han incluido en la extensión "default" que está implícitamente disponible en cualquier secuencia de comandos. `{$USE default}` es posible pero redundante. Usar la [directiva](../coding/directives.md#$cleo) `{$CLEO}` implica `{$USE CLEO}`, por lo que los opcodes estándar de CLEO estarán disponibles para el compilador. ## Lista de Extensiones Sanny Builder incluye algunas extensiones comunes y reconocidas para diferentes modos de edición:
