Un **tipo enumerado** o **enum** es un conjunto finito de constantes (llamados _campos de enumeración_) bajo un prefijo común (_nombre de enumeración_). Los tipos enumerados tienen algunas ventajas sobre las constantes ordinarias. No contaminan el espacio de nombres global y son ciudadanos de primera clase en el sistema de tipos. Los parámetros con un tipo enumerado están limitados en el rango de valores permitidos, lo que a su vez ayuda a evitar errores causados por un uso de ID no válido. **Enums.txt** es un archivo de texto que define una lista de los tipos enumerados disponibles en el [modo de edición](README.md). ## Formato del archivo Cada tipo enumerado se define con la siguiente sintaxis: ``` enum <nombre> <campos> end``` `nombre` - cualquier combinación de letras, dígitos y `_` \ `campos` - una o varias líneas con el formato: - `<nombre del campo> [= <enum field value>]` - donde - `nombre del campo` - cualquier combinación de letras, dígitos y `_` - `valor del campo` - un número o una [cadena literaria](../coding/data-types.md#string-literals) entre comillas dobles. si no está presente, el compilador asigna un [nuevo valor](enums.txt.md#enum-values) automáticamente. ``` enum Town LS = 0 SF = 1 LV = 2 end ``` Los campos de enumeración se pueden separar con saltos de línea (consulte el ejemplo anterior) o con una coma: ``` enum Town LS = 0, SF = 1, LV = 2 end ``` Es posible definir muchos tipos enumerados en un archivo. Cada enumeración debe tener un nombre único. :::warning No se permiten comentarios ni ninguna sintaxis adicional en el archivo de enumeraciones. ::: ### Campos de Enumeración Cada campo de enumeración obtiene un valor implícita o explícitamente. Se asignan valores explícitos cuando una combinación del signo igual y el valor sigue al nombre del campo. Sanny Builder permite utilizar números enteros y cadenas literales como valores de enumeración: ``` enum Town LS = 0 SF = 1 LV = 2 end ``` `Town.LS` es igual a `0`, `Town.SF` es igual a `1` y `Town.LV` es igual a `2`. ``` enum TicTacToe Player1 = "X" Player2 = "O" end ``` `TicTacToe.Player1` es igual a `X` y `TicTacToe.Player2` es igual a `O`. Los valores implícitos se asignan automáticamente a partir de `0` para el primer campo, `1` para el segundo campo, `2` para el tercero y así sucesivamente: ``` enum Town LS SF LV end ``` `Town.LS` es igual a `0`, `Town.SF` es igual a `1` y `Town.LV` es igual a `2`. Para una combinación de valores implícitos y explícitos, se aplican las siguientes reglas: * un campo que sigue a un valor explícito se asigna a ese valor incrementado en `1` ``` enum E A = 10 B C = 100 D E end ``` `E.A` = `10`, `E.B` = `11`, `E.C` = `100`, `E.D` = `101`, `E.E` = `102` * un campo que sigue a un valor de cadena explícito obtiene su valor después del nombre del campo: ``` enum Month Jan = "January" Feb = "February" March end ``` `Month.Jan` = `"January`, \ `Month.Feb` = `"February`, \ `Month.March` = `"March"` :::warning Una enumeración no puede tener una combinación de valores enteros y de cadena. El primer valor explícito toma el control de qué tipo de valores obtiene esta enumeración: ``` enum Int X, Y // X = 0, Y = 1, end enum String X, Y = "Y" // X = "X", Y = "Y" end ``` :::
