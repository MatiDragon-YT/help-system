CLEO 3 permite agregar nuevos scripts innecesariamente para comenzar un nuevo juego. Estrictamente hablando, el estado de main.scm no importa para estos scripts, ya que los scripts CLEO se ejecutan y funcionan por separado. ¿Qué da? Puedes agregar un nuevo script en cualquier etapa del juego o eliminar uno existente. Aun así, no necesita editar main.scm, por lo que la necesidad de comenzar un nuevo juego, la incompatibilidad de guardados, los límites de scripts: todos estos problemas desaparecen de inmediato. El CLEO-script es un script ordinario escrito usando los opcodes en Sanny Builder. Cada script debe hacerse como un solo archivo con una extensión .cs (script personalizada). El requisito dominante para escribir dichos scripts es que deben compilarse como scripts externos. Hay varias formas de hacer que un CLEO-script funcione correctamente en el juego. Primero: agrega un nuevo script externo en el archivo fuente, lo compila y luego extrae ese script del archivo script.img. El archivo extraído tendrá la extensión .scm, por lo que solo necesita cambiarlo en .cs y copiarlo en la carpeta 'game\CLEO'. La segunda forma es más fácil. Cree un nuevo archivo en Sanny Builder, escriba la [directiva $E](../coding/directives.md#external) al principio y compílelo. El archivo compilado será el mismo que el extraído del script.img. Solo necesita cambiar su extensión en .cs y copiar en el 'juego\CLEO'. Por último, el más fácil. Cree un nuevo archivo en Sanny Builder, escriba la [directiva $CLEO](../coding/directives.md#cleo) al principio del archivo y compile. El archivo compilado se copiará en el directorio 'game\CLEO' y tendrá una extensión de archivo especificada por la directiva (.cs por defecto). Los CLEO-scripts podrían usarse como misiones. El proceso de compilación es el mismo, pero el archivo de salida debe tener la extensión .cm (Custom Mission). Para iniciar esta misión, use el [opcode 0A94](opcodes.md#0A94). Las reglas que debes seguir al utilizar CLEO 3: 1. **Un archivo - un hilo**. Cada CLEO-script debe hacerse como un solo hilo compilado en un solo archivo. Si necesita más hilos para su script, use los opcodes 004F o [0A92](opcodes.md#0A92), dependiendo de dónde se encuentre ese subproceso adicional: en main.scm o en un archivo cs separado. 2. **Nunca use el opcode 004E en los hilos CLEO**. Funcionan por separado de los hilos comunes, por lo que el uso de este opcode provocará un error. Use el opcode 0A93, si necesita detener un hilo CLEO. Por el contrario, nunca use [el opcode 0A93](opcodes.md#0A93) en las misiones CLEO, sino que use el 004E para cancelar una misión. 3. **Nunca use variables globales personalizadas en sus CLEO-scripts ($car, $foo, $blahblah, etc.)**, o su script funcionará incorrectamente. Puede usar solo unas pocas variables globales (lea a continuación). Para instalar el script CLEO solo es necesario copiar el archivo con extensión .cs a la carpeta ‘game\CLEO’. El nombre del archivo no importa, pero se utiliza en la inicialización del hilo para nombrarlo. Por ejemplo, el hilo CLEO que se encuentra en el archivo showsavescreen.cs tiene el nombre SHOWSAV. Para eliminar el script, simplemente elimine el archivo adecuado del directorio CLEO. El estado del hilo CLEO se puede guardar y cargar junto con los hilos comunes. No se guarda por defecto. Los archivos guardados se encuentran en la carpeta CLEO\cleo_saves. Es un guardado propio para cada ranura de guardado. El archivo guardado almacena solo los hilos CLEO que tienen el indicador de guardado habilitado ([opcode 0A95](opcodes.md#0A95)). El archivo guardado almacena no solo los hilos activos, sino también los que se ven obligados a finalizar ([opcode 0A93](opcodes.md#0A93)), para evitar que se vuelvan a ejecutar al cargar. Si al cargar uno de los hilos guardados no se encuentra, no hay error. Si agrega un nuevo script que no está presente en el archivo guardado actual, se cargará junto con los demás y funcionará desde el principio. El hilo CLEO es el hilo ordinario en general. Tiene 32 variables locales y 2 temporizadores que se incrementan constantemente. Puede pasar hasta 34 parámetros en la creación de ese hilo o una misión CLEO (por analogía con el opcode 004F). Una cosa importante se refiere a la comunicación de datos entre un script CLEO y main.scm. Para usar el código SCM, sus variables, hilos, debe compilar el script CLEO como uno externo (primera forma). Pero no se recomienda este método, ya que dicho script se vuelve muy sensible a cualquier cambio de main.scm. *Implícitamente, un script CLEO debería ser suficiente por sí mismo y usar un mínimo de datos SCM*. Sin embargo, puede usar libremente nombres de variables globales de CustomVaribles.ini (por ejemplo, `$PLAYER_CHAR`, `$PLAYER_ACTOR`, `$ONMISSION`, etc.). Además, para la comunicación de datos entre main.scm y CLEO, o entre los scripts de CLEO, puede usar las variables CLEO (consulte los opcodes 0AB3, 0AB4). El conjunto completo de CLEO incluye ejemplos de scripts simples con fuentes.
