Aquí puede configurar nombres de archivos de origen, formato de etiquetas, mayúsculas y minúsculas para cadenas literales, etc. :::l ![](../../img/options-format-en.png) ::: ## Formato del Nombre de la Fuente (source) Formatos para archivos producidos por el desensamblador o el compilador. :::l ![](../../img/formats-files-en.png) ::: El campo superior contiene la ruta completa y el nombre de un archivo de texto creado durante el desmontaje. Dos campos a continuación contienen la ruta y el nombre de los archivos binarios `SCM` e `IMG` (si el [modo de edición](../../edit-modes/README.md) es `GTA SA`) creado durante la compilación . Todos los campos comparten la misma sintaxis. También pueden usar palabras especiales que representen partes del nombre del archivo de origen:\ \ `$dir` - la ruta del archivo\ `$name` - el nombre del archivo\ `$ext` - la extensión del archivo\ \ P.ej. si desensambla el archivo `C:\MyDir\main.scm`, entonces\ \ `$dir` es `C:\MiDir`\ `$name` es `main`\ `$ext` es `.scm`\ \ Si el formato es `$dir\$name.txt`, el archivo de salida se llama `C:\MyDir\main.txt`. ## Formato de las Etiquetas Formatos para nombres de etiquetas usados durante el desensamblaje. :::l ![](../../img/formats-labels-en.png) ::: ### Posición Global El nombre de una etiqueta es numérico, p. `@12345`. El número es el desplazamiento de la etiqueta desde el principio del archivo de origen. ### N. del Bloque+Pos. Local Un nombre de etiqueta incluye un nombre de la secuencia de comandos donde se encuentra la etiqueta (como se define con el comando `script_name`) y el desplazamiento desde el principio de la secuencia de comandos, p. `@MAIN_12`. ### N. del Bloque+Pos. Global Un nombre de etiqueta incluye un nombre de la secuencia de comandos donde se encuentra la etiqueta (como se define con el comando `name_thread`) y el desplazamiento desde el principio del archivo de origen, p. `@MAIN_12345`. {% hint style="info" %} Si desea que las compensaciones sean hexadecimales, seleccione la casilla de verificación en la parte inferior. {% endhint %} ## Nombres Propios de... Estas opciones indican al desensamblador que use nombres personalizados definidos por el usuario en los archivos externos. :::l ![](../../img/formats-custom-names-en.png) ::: El archivo [`CustomLabels.ini`](../../edit-modes/customlabels.ini.md) contiene la lista de nombres de etiquetas y sus compensaciones. Si el desensamblador encuentra una coincidencia entre el desplazamiento de la etiqueta en el archivo fuente y el desplazamiento definido en el archivo INI, le da a esta etiqueta un nombre asociado con este desplazamiento. El archivo `CustomVariables.ini` contiene la lista de direcciones de [variables globales](../../coding/variables.md#global-variables) y sus nombres personalizados. El desensamblador usa este archivo para nombrar variables globales. El archivo `CustomArrays.ini` contiene los nombres de las arreglos en la siguiente sintaxis: el primer número es la dirección de la variable global que es el primer elemento del arreglo, luego el tamaño del arreglo, luego un nombre personalizado. Ayuda al desensamblador a reconocer [elementos del arreglo](../../coding/arrays.md#using-constant-indexes). ## Usar el Caso Tipográfico :::l ![](../../img/formats-case-en.png) ::: Esta opción tiene un doble propósito. El desensamblador nombra [variables globales](../../coding/variables.md#variables-globales), [etiquetas](../../coding/data-types.md#etiquetas), [arreglos]( ../../coding/arrays.md) utilizando las mayúsculas y minúsculas seleccionadas. El compilador escribe [literales de cadena](../../coding/data-types.md#string-literals) usando también el caso seleccionado.
