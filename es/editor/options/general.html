Esta pestaña contiene la configuración general utilizada por Sanny Builder. :::l ![](../../img/options-general-en.png) ::: ## Directorio del Juego :::l ![](../../img/main_dir.png) ::: Sanny Builder necesita ciertos archivos del juego mientras trabaja, por lo que debe elegir un directorio correcto donde está instalado el juego para el modo de edición actual. Puede ingresar la ruta manualmente o seleccionando el directorio a través del cuadro de diálogo que aparece cuando presiona el botón. ## Idioma de la Interfaz :::l ![](../../img/general-language-en.png) ::: Lista de traducciones disponibles para Sanny Builder. Seleccione un idioma para actualizar la interfaz del programa. Los archivos de traducción se encuentran en la carpeta `lang`. Puede encontrar más información sobre cómo crear una nueva traducción o actualizar la existente en [sannybuilder/translations](https://github.com/sannybuilder/translations). ## Asociación de Archivos :::l ![](../../img/sb-options-associate.png) ::: Puede asociar diferentes extensiones de archivo con Sanny Builder. Cuando se selecciona una casilla de verificación, su Registro de Windows cambia y puede abrir los archivos con esta extensión simplemente haciendo clic en él. Para eliminar la asociación, anule la selección de la casilla de verificación. ## Configuraciones Principales :::l ![](../../img/main_opt.png) ::: ### Sobrescribir Siempre el Archivo de Salida Esta opción determina cómo el desensamblador trata el archivo de salida cuando ya existe un archivo con el mismo nombre. De manera predeterminada, el desensamblador conserva el archivo existente y crea uno nuevo con el número adicional en el nombre (p. ej. `main[0].txt`). Cuando esta opción está marcada, el desensamblador reemplaza el archivo existente con un nuevo archivo. ### Mostrar Progreso Muestra una barra de progreso en vivo durante el desmontaje o la compilación. Mostrar el progreso ralentiza ligeramente el tiempo de ejecución. Cuando esta opción está desactivada, la barra de estado muestra las palabras: `Por favor, espere`. ### Mostrar Reporte Muestra una ventana emergente con diversa información sobre el archivo compilado. ### Mostrar Alerta Esta opción solo se usa durante la compilación de los scripts de San Andreas. Si el juego se está ejecutando, el archivo `script.img` que contiene scripts externos no se puede sobrescribir ya que el juego usa este archivo y el compilador se queja al respecto. Puede desactivar la alerta desmarcando esta casilla. ### Abrir IMG Manualmente Cuando comienza el proceso de desmontaje, el programa busca en el archivo `script.img` que contiene algunos scripts de juegos. Si este archivo no está presente en la misma carpeta con el archivo `.SCM` o en la carpeta `San Andreas\data\scripts`, se muestra el mensaje de error. Si esta opción está habilitada, aparece un cuadro de diálogo de selección de archivo y puede proporcionar otro archivo `script.img` manualmente. ### Cargar Juego Rápidamente Con esta opción, Sanny Builder omite las pantallas de inicio al iniciar San Andreas (`F8`). Si esta función no funciona con la versión de su juego, deshabilite esta opción. ### Chequeo de Condiciones Esta opción tiene dos significados. Primero, el programa reemplaza el [número de condiciones](../../coding/conditions.md#syntax) en el opcode `IF` con `AND` o `OR`. En segundo lugar, el programa puede compilar la construcción `IF..END` y calcular el parámetro `IF` necesario en las condiciones con las palabras `AND` u `OR`. ### Chequeo de Rangos El número de [variables](../../coding/variables.md) locales y globales es limitado. Cuando esta opción está marcada, el compilador verifica si una variable se ajusta al rango disponible. ### Escribir Opcodes Si esta opción no está marcada, el desensamblador usa [clases](../../coding/classes.md) disponibles y [palabras clave](../../coding/keywords.md) en lugar de opcodes. Además, las expresiones matemáticas simples no tienen códigos de operación. Si la opción está marcada, todos los códigos de operación están presentes en el archivo de salida. ### Reemplazar Números de Misiones Cuando esta opción está marcada, el desensamblador [reemplaza los números de misión](../features.md#replaceing-mission-numbers-with-their-names) en `start_mission` con sus nombres. El nombre de la misión es el nombre de la etiqueta definido en el header(encabezado) del archivo. Este nombre también podría usarse para navegar rápidamente al código de misión. ### Insertar Nombre Original de Misión Cuando esta opción está marcada, el desensamblador agrega el [título de la misión](../features.md#custom-mission-titles) como un comentario para el opcode `start_mission` y para la línea `DEFINE MISSION` en el header del archivo. ### Agregar información adicional a SCM Si esta opción está marcada, el compilador agrega información adicional al final del archivo resultante. Esta información se usa más tarde cuando este archivo se desarma para restaurar la fuente más cerca del original. Se almacenan los siguientes datos: * constructor de posiciones [HEX..END](../../coding/hex..end.md)¿ * nombres de [variables globales](../../coding/variables.md#global-variables) * código fuente completo (use [$NOSOURCE](../../coding/directives.md#usdnosource) para desabilitada) * [modo de edición](../../edit-modes/README.md) actual {% hint style="info" %} El archivo compilado con la información adicional no se puede abrir en Sanny Builder antes de la versión 3.00 (a menos que la [opción](../console.md#ignore_unknown) de `ignore_unknown` esté habilitada). A partir de v3.8.0, el desensamblador puede [ignorar información adicional](../console.md#skip_extra_info). {% endhint %}
