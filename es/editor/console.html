La **consola** ofrece la posibilidad de cambiar las opciones de depuración de Sanny Builder. En su mayoría, se relacionan con las funciones del desensamblador y puede ver los cambios al desensamblar un archivo. La ventana de la consola se abre después de presionar `Ctrl+Ñ`. Puede ingresar los comandos en la línea de entrada. Después de escribir un comando, presione `Enter` para continuar. La consola mostrará un resultado. :::l ![](../img/console.png) ::: ## Comandos Disponibles Los comandos tiene tipografía insensible. `help` – muestra la información de ayuda sobre la consola\ `clear` – limpia la consola\ `list` – muestra en la consola la lista de todos los comandos\ `list options` – enumera todas las opciones de depuración disponibles\ `toggle [on/off]` – habilitar o deshabilitar [opciones de depuración](console.md#debug-options) The input command ends either with the word `ON` (the option will be enabled) or `OFF` (the option will be disabled), or nothing (the option will be toggled from `ON` to `OFF` or from `OFF` to `ON`). {% hint style="info" %} Antes de habilitar las siguientes opciones, active el modo de depuración con el comando `toggle debug_mode ON` {% endhint %} ## Opciones de Depuración ### CODE_OFFSETS El desensamblador imprime el desplazamiento de cada comando. ### VAR_COUNTER Después de compilar, la consola contiene la lista de las [variables globales](../coding/variables.md#variables-global) utilizadas solo una vez en el código (es decir, se pueden eliminar o reemplazar con [variables locales](../coding/variables.md#variables-locales)). ### IGNORE_UNKNOWN El desensamblador ignora los códigos de operación desconocidos, los parámetros incorrectos, etc. Ayuda a abrir casi cualquier archivo que solía estar protegido o compilado incorrectamente. ### CONSTANT_INDEXES El desensamblador imprime los elementos de la Array como variables globales con índices. Está disponible para juegos `GTA SA`, `LCS`, `VCS`. P.ej. dada una Array de tres elementos que comienzan en `$10`, esta opción afecta la forma en que se ven las variables después del desmontaje: |Sin `CONSTANT_INDEXES`|Con `CONSTANT_INDEXES`| |-| |$10|$10[0]| |$11|$10[1]| |$12|$10[2]| ### SKIP_SCM_HEADER Desensamblador omite el encabezado del archivo de entrada. Permite abrir scripts sin cabeza (por ejemplo, los de `script.img` o CLEO scripts). Con esta opción, el compilador crea archivos `.scm` sin el encabezado (similar a la [directiva](../coding/directives.md#$external)`{$EXTERNAL}`). ### SKIP_EXTRA_INFO El desensamblador ignora la sección [información adicional](options/general.md#add-extra-info-to-scm) adjunta al archivo de entrada. También trata esta sección como un conjunto de instrucciones regulares de SCM, por lo que se recomienda habilitar la opción `IGNORE_UNKNOWN`. ## Formas alternativas Sanny Builder ofrece algunas otras formas de alternar las opciones de depuración fuera de la consola que pueden ser útiles en algunos casos. ### Corriendo con --debug La [opción](cli.md#debug) `--debug` proporciona una forma alternativa de cambiar las opciones de depuración. Ejecute Sanny con el parámetro `--debug X`, donde `X` es una serie de `0` y `1`. Cada dígito de la serie corresponde a una opción de depuración particular: |Indice|Opción de Depuración| |-| |1|CODE_OFFSETS| |2|IGNORE_UNKNOWN| |3|VAR_COUNTER| |4|CONSTANT_INDEXES| |5|SKIP_SCM_HEADER| |6|SKIP_EXTRA_INFO| ``` sanny.exe --debug 110000 ``` El primer `1` habilita la opción `CODE_OFFSETS`, el segundo `1` habilita el modo `IGNORE_UNKNOWN`. Las opciones restantes están deshabilitadas. ### Usando la GUI Al hacer clic en el botón más a la derecha en la barra de herramientas principal, se muestra un menú desplegable con la lista de opciones de depuración. :::center ![](../img/sb-debug-options-dropdown.png) ::: Cada elemento del menú alterna una opción de depuración particular. El cambio entra en vigor inmediatamente.
