Ciertos elementos de código deben comenzar o terminar con caracteres específicos. Ayudan al compilador a reconocer su significado. La siguiente lista contiene todos los tipos de datos conocidos por el compilador. ## Variables Una [variable global](variables.md#global-variables) comienza con `$` seguido de un identificador válido (cualquier combinación de letras, dígitos y `_`). ```sb3 0004: $MyVar = 100 ``` Una [variable local](variables.md#local-variables) como nombre solo puede tener un número seguido de `@`. El número es el índice de variable local en la lista de variables locales [asignadas](../scm-documentation/gta-limits.md) a este script o una misión. ```sb3 0006: 100@ = 10 ``` Una variable `ADMA` (**A**dvanced **D**irect **M**emory **A**ccess) es una referencia al desplazamiento en el archivo `main.scm`. Sirven como variables globales, es decir, se puede leer y escribir en una dirección en `main.scm`. ```sb3 $myVar = &0 // leer los primeros 4 bytes de main.scm (&0) y escribir el valor en $myVar &57 += &120(&231,4i) // Se puede utilizar como elemento de un arreglo ``` Las variables `ADMA` no afectan al tamaño del espacio de variables globales en el encabezado `main.scm`. ## Etiquetas Las etiquetas se utilizan para hacer referencia a la ubicación del código desde incondicionales (opcode `0002`, `0050`) o saltos condicionales (opcode `004D`). Empiezan con `:` seguido de un identificador válido (un nombre de etiqueta). ```sb3 :MyLabel ``` Para referenciar la etiqueta desde un opcode, utilice el `@` y, a continuación, escriba el nombre de la etiqueta. ```sb3 0002: jump @MyLabel ``` Si el nombre de una etiqueta se escribe como una declaración independiente seguida de `()` representa una llamada de subrutina con el comando `gosub`: ```sb3 // Las siguientes declaraciones son equivalentes MyLabel() gosub @MyLabel ``` ## Cadenas literarias Un texto encerrado entre comillas simples `''` es un literal de cadena corta (`15` caracteres como máximo). ```sb3 03A4: script_name 'MAIN' ``` También se permiten las cadenas en blanco: `' '`. Un texto encerrado entre `" "` es un literal de cadena larga (la longitud máxima está determinada por el código de operación que usa) Si el literal contiene `"` debes escribir `\` antes. ```sb3 0662: write_debug_message "Hello, \"world\"! \n 'Here we go!'" ``` También se permiten las cadenas en blanco: `" "`. ## Variables de Cadenas Una variable global que contiene una cadena corta se inicia con `s$`. ```sb3 05A9: s$MyString = 'GLOBAL' ``` Una variable local que contiene una cadena corta se inicia con `@s`. ```sb3 05AA: 1@s = 'LOCAL' ``` Una variable global que contiene una cadena larga se inicia con `v$`. ```sb3 06D1: v$MyString = "LONG_GLOBAL" ``` Una variable local que contiene una cadena larga se inicia con `@v`. ```sb3 06D2: 1@v = "LONG_LOCAL" ``` ## Nombres de Modelos Los modelos definidos en los archivos `.ide`, pueden ser referenciados con `#` seguido de su nombre. ```sb3 0247: request_model #CELLPHONE ``` ## Numeros Hexadecimales `0x` - positivo\ `-0x` - negativo ```sb3 0004: $var = -0xBB08 ``` Un número hexadecimal debe estar dentro del rango `-80000000..7FFFFFFF`.
