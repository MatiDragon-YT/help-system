Main.scm'nin mimarisi Şimdi orijinal main.scm ve script.img'yi yeni bir kodlama çalışma klasörüne kopyalayın (veya istediğiniz herhangi bir yer: masaüstü, Belgelerim, geri dönüşüm kutusu, annenizin belgeleri 😁 ....) oluşturdunuz ve şimdi açınız (yedekleme yapın !!!!! orijinal main.scm ve script.img için hemen yeni bir klasör oluşturun) aç'a tıklayarak ve SCM Dosyasını seçin (*.scm). İlerleme çubuğunun dolduğunu görebilirsiniz (Bu nedenle "İlerlemeyi göster" seçeneğini kapatalım). #### Tanım listesi bölümü: Açtığınızda ise şunu göreceksiniz:\ **KOD** ```sb3 // This file was decompiled using sascm.ini published by Seemann (http://sannybuilder.com/files/SASCM.rar) on 13.10.2007 DEFINE OBJECTS 389 DEFINE OBJECT SANNY BUILDER 3.03 DEFINE OBJECT INFO // Object number -1 DEFINE OBJECT KEYCARD // Object number -2 DEFINE OBJECT AD_FLATDOOR // Object number -3 DEFINE OBJECT KB_BANDIT_U // Object number -4 etc... ``` Gördüğünüz gibi main.scm içinde kullanılacak nesneleri tanımlıyor, üstte söylendiği gibi 389 tane var (DEFINE OBJECTS 389). Bir nesne eklemek için sonuncuya gidin ve kendinizinkini ekleyin. Daha aşağıda (çok daha aşağıda) bunu bulabilirsiniz:\ **KOD** ```sb3 DEFINE MISSIONS 135 DEFINE MISSION 0 AT @INITIAL DEFINE MISSION 1 AT @INITIL2 DEFINE MISSION 2 AT @INTRO DEFINE MISSION 3 AT @NONE etc... ``` Bu, görevleri tanımlar, üstte söylendiği gibi 135 tane var. (`DEFINE MISSIONS 135`). Xbox, Playstation 2 veya [jarjar 2-player](http://www.gtaforums.com/index.php?showtopic=208450&view=findpost&%20%20p=3069233) mod dosyalarında 2 oyunculu konsol görevi olan daha fazla görev olduğunu görebilirsiniz ama bu başka bir hikaye...\ \ **KOD** ```sb3 DEFINE EXTERNAL_SCRIPTS 78 // Use -1 in order not to compile AAA script DEFINE SCRIPT PLAYER_PARACHUTE AT @PLCHUTE // 0 DEFINE SCRIPT PARACHUTE AT @PARACH // 1 DEFINE SCRIPT BCESAR2 AT @BCESAR2 // 2 DEFINE SCRIPT BCESAR3 AT @COKEC // 3 DEFINE SCRIPT SLOT_MACHINE AT @BANDIT // 4 etc... ``` Bu, gerektiğinde tetiklenen komut dosyaları (programlar) olan harici komut dosyalarını tanımlar, yukarıda belirtildiği gibi 78 tane vardır. Paraşütü idare eden şey bu (player -`PLAYER_PARACHUTE`- and non-players -`PARACHUTE`-), Kumarhanedeki oyunlar (`ROULETTE`, `VIDPOK` -Video Poker-, `BLACKJ` -Black Jack-, `WHEELO` -Şans Çarkı-, etc...), Dükkanlar (`AMMU`, `TATTOO`, `BARBER`, `JUNKFUD`, `CARMOD`), ve birçok değişik şey yada çokta değişik olmayan (`DEBT`, `STRIPW`, `GF_SEX`, `HOME_BRAINS`...) şeylerde var ama sonra anlatacağım bunları. Sonra bu 2 satıra geliyorsunuz:\ **CODE** ```sb3 DEFINE UNKNOWN_EMPTY_SEGMENT 0 DEFINE UNKNOWN_THREADS_MEMORY 574 ``` Bunlar belgelenmemiş ve bizim için yararsız. #### ANAYAPIT (MAIN) kısmı: Now you can see something like this:\ **CODE** ```sb3 {$VERSION 3.1.0027} //-------------MAIN--------------- 03A4: name_thread 'MAIN' 016A: fade 0 time 0 042C: set_total_missions_to 147 030D: set_max_progress 187 0997: set_total_respect_points_to 1339 01F0: set_max_wanted_level_to 6 0111: set_wasted_busted_check_to 0 00C0: set_current_time_hours_to 8 current_time_minutes_to 0 04E4: unknown_refresh_game_renderer_at 2488.562 -1666.864 03CB: set_rendering_origin_at 2488.562 -1666.864 13.3757 062A: change_stat 165 to 800.0 // floating-point values 062A: change_stat 23 to 50.0 // floating-point values 062A: change_stat 21 to 200.0 // floating-point values 062A: change_stat 160 to 0.0 // floating-point values 0629: change_stat 181 to 4 // integer values 0629: change_stat 68 to 0 // integer values 0053: $PLAYER_CHAR = create_player #NULL at 2488.562 -1666.864 12.8757 etc... ``` **Oraya** kadar (Sanny arama aracını kullanın ve "görev 0"ı arayın):\ **CODE** ```sb3 //-------------Mission 0--------------- // Originally: Initial 1 :INITIAL 03A4: name_thread 'INITIAL' 06C8: enable_riot 0 0004: $1515 = 0 0005: $1500 = 5.0 etc... ``` Ana bölüm veya ana bölüm olarak adlandırılır (ve bazen ana iş parçacığı olarak adlandırılan belirsizlikle). Bunda, örneğin görevleri başlatan kırmızı daireleri oluşturan, telefon görüşmelerini yöneten şey ('CELLFON' ve diğer şeyler), grilfriends'i yöneten, her şeyin tetiklendiği (başlatıldığı) yer olan pek çok küçük komut dosyası vardır. birçok mod eklendi ve kodlamaya buradan başlayacaksınız. Ana bölüm birçok küçük iplikten oluşur, ancak ana konu ile başlar. Ve iş parçacığı (ve tek iş parçacığı) motorun kendisi tarafından oluşturulur, onu başlatmak için diğerleri gibi "004F" (**create_thread**) iş parçacıkları oluşturan işlem kodunu kullanmanız gerekmez. Endişelenmeyin, daha sonra tüm bölüm konulara ayrılacaktır. Ana bölümde yalnızca yerel değişkenleri kullanabilirsiniz ('0@', '1@', '2@', vb.) '31@' dahil, '32@' ve '33@' dahili değişkenlerdir sizin de kullanabileceğiniz yerel zamanlayıcılar (geçen süreyi sayar, daha sonra göreceğiz). Ancak `33@` üzerindeki herhangi biri ana kısımda çalışmaz. Sanny mutlu olmayacak ve size yerel var olduğunu söyleyecektir. menzil dışında (bu yüzden "Menzil kontrolü" seçeneğini açtık). İstediğiniz küreselleri kullanabilirsiniz ancak sayıları 16383 ile sınırlıdır, yeterince uzak olmalıdır. #### Şimdide görev bölümü: Then after:\ **CODE** ```sb3 //-------------Mission 0--------------- // Originally: Initial 1 :INITIAL 03A4: name_thread 'INITIAL' 06C8: enable_riot 0 0004: $1515 = 0 0005: $1500 = 5.0 etc... ``` Bu ilk görevdir (aslında bir görev değildir çünkü yeni bir oyunun ara sahneden önceki ilk milisaniyelerinde yapılır ve herhangi bir görev puanı vermez). Bu görev aslında park halindeki arabaları yaratıyor, bazı silahlar toplanıyor ve başka şeyler. Görev 1'e ulaşıldıktan sonra (Görev 1'i bulun)\ **CODE** ```sb3 //-------------Mission 1--------------- // Originally: Initial 2 :INITIL2 03A4: name_thread 'INITIL2' 0004: $3407 = 25 0004: $3408 = 100 0004: $3409 = 250 etc... ``` Görev 0 ile aynı, ama bu bir görev değil, sadece bazı değişkenleri sayı değerine eşitliyor, bazı arabaları tekrar park ediyor, bazı kapıları ve asansörleri çalıştırıyor. Şimdi ise Görev 2\ **CODE** ```sb3 //-------------Mission 2--------------- // Originally: Intro :INTRO 03A4: name_thread 'INTRO' 0050: gosub @INTRO_47 00D6: if 0112: wasted_or_busted // mission only 004D: jump_if_false @INTRO_38 0050: gosub @INTRO_9715 etc... ``` Bu, giriş ara sahnesinin bir parçasıdır ve görev, CJ'nin evindeki (Ganton'daki Johnson House) kırmızı daireye geldiğinizde sona erer. Oyun salonlarından sonra oyun salonları gelir (*orijinal olarak: Video oyunu: Uranüs'ten süründüler; orijinal olarak: Video oyunu: Duality; orijinal olarak: Video oyunu: Go Go Space Monkey; orijinal olarak: Video oyunu: BUMBLE'a hazırlanalım; orijinal olarak: Video oyunu : poker*) sonra bilardo (*başlangıçta: Havuz*), dans (başlangıçta: Lowrider (bahis ve dans)) ve Sıfır "gizli/bonus" görevi (*başlangıçta: Beefy Baron*) gibi diğer şeyler ve bunlardan sonra, " Oyunun ilk görevi olan Big Smoke" Görevi (görev 11). Ve sonra görevin 134. sonuna kadar böyle. Eğer bir görev eklemek istiyorsanız, bundan sonra ekleyin ve görev tanımla bölümünde tanımlayın. (devamı daha sonra). #### Ekstradan Scriptler (External Scripts) bölümü (son bölüm) Son görevden sonra, ilk `PLAYER_PARACHUTE` ile başlayan tanımlanmış tüm harici komut dosyasını görebilirsiniz.:\ **CODE** ```sb3 //-------------External script 0 (PLAYER_PARACHUTE)--------------- :PLCHUTE 03A4: name_thread 'PLCHUTE' 0247: request_model #GUN_PARA :PLCHUTE_16 etc... ``` ve ondan sonra, **CODE** ```sb3 //-------------External script 1 (PARACHUTE)--------------- :PARACH 03A4: name_thread 'PARACH' 0004: $8275 = 0 etc... ``` ve böylece 77. harici komut dosyalarına kadar. Harici komut dosyalarındaki birçok şeyin, harici ismine bakarak, genellikle ne için kullanıldığını anlayabilirsiniz. Şimdilik en önemli kısım ana kısım. 'Ana bölümün' ne kadar süredir derlendiğini bilmek ve rapora bakmak için size "ANA boyut: XXXXXX b" diyor ve ardından en büyük görev olan "En Büyük Görev: XXXX b." ve En Büyük harici komut dosyası : :::l ![](mcp1/retfyhj2.png) ::: Bu nedenle "Raporu göster" seçeneğini tikledik. ### 2. Fix Paket bir SCM ve Kaynak Kodları #### A. Fix Paket Bir SCM İlk kodlama projeleriniz için küçük (orijinal olan çok büyük) ve komut dosyası yazmak için yararsız hiçbir şeye sahip olmayan bir main.scm'ye ihtiyacınız var. Derleme yaparken zaman kazanacaksınız ve ilk ara sahneyi beklemek zorunda kalmayacaksınız. Ayrıca scm sınırları nedeniyle yukarıyı okuyun. Yani fix paket bir scm, herhangi bir görev, herhangi bir harici komut dosyası, herhangi bir ana iş parçacığı içermeyen bir main.scm'dir. Neredeyse hiçbir şey içermez, sadece oynatıcıyı oluşturur. Sanny Builder'ı kurduğunuz \data\sa klasöründe stripped.txt adıyla bir tane bulabilirsiniz, garip görünüyor çünkü Sanny Builder sınıfları sözdiziminde yazılmış, size bu konuda fazla bir şey anlatmayacağım, daha fazlasını Sanny Builder'ın dosyasında bulabilirsiniz Yardım. İyi olmadığından değil çünkü önce "opcode yöntemi" ile öğrenmenin daha kolay olacağını düşünüyorum. Böyle bir şeye sahip olmalı: **CODE** ```sb3 { use macro (Ctrl+J) "headsa" to insert a file header } {$VERSION 3.0.0000} var $PLAYER_CHAR: Player end // var 03A4: name_thread 'MAIN' 01F0: Kaynak Kodları Çoğu satırın başında 4 basamaklı bir sayı ve ardından 0001:, 087B:, 016A: veya 004E: gibi iki nokta vardır, bunun gibi 0605: Sanny Builder'da. Bunlara işlem kodları (kaynak kodları) denir !!!!!! İşte bu yüzden "İşlem kodlarını yaz" seçeneğini belirledik. İşlem kodu, oyun motorunun verilen parametrelerle yapacağı komutun (eylem veya işlem) sayısıdır, tüm işlem kodlarının (birkaç tanesi hariç) sabit sayıda parametresi vardır. Parametreler (Paramlar veya yalnızca P1, P2 veya %p olarak da adlandırılır) komutun girdileridir (sayılar, aktörler, araçlar, metin dizileri veya herhangi bir değişken türü). Sol alttaki Sanny oluşturucu size bir işlem kodunun ne kadar parametreye ihtiyacı olduğunu söyler: :::l ![](mcp1/yrrgrg.png) ::: İşte iki örnek: **simple**\ **CODE** ```sb3 000A: 3@ += 1 // integer values ``` * `000A` : işlemi yapan işlem kodu tamsayılara ekler. * `3@` and 1 : parametrelerdir, bu işlem kodunun 2 parametresi vardır. Böylece bu yerel değişkeni (daha sonra) `3@` alır ve ona bir ekler, sanki `3@` 1'e eşitse, şimdi 2'ye eşit olacaktır !!! ```sb3 0605: actor $PLAYER_ACTOR perform_animation_sequence "ATM" from_file "PED" 4.0 loop 0 0 0 0 6000 ms ``` * `0605` : the opcode of the command * `$PLAYER_ACTOR`, `"ATM"`, `"PED"`, `4.0`, `0`, `0`, `0`, `0`, `6000` : are the parameters, this opcode has 9 parameters. Bu komut, mevcut bir aktörün (karakter) `$PLAYER_ACTOR` animasyonunu `"PED"` animasyon dosyasında `"ATM"` gerçekleştirmesini sağlar, `4.0` animasyonun oynatma hızıdır, döngü 0, yapmayacağı anlamına gelir anim tekrar bittiğinde, eğer tamsayı 1'e ayarlanmışsa, oyuncu başka bir şey yapması (veya yok etmesi veya "özgür" olması söylenene kadar) bunu tekrar tekrar yapacaktır (döngü) - ekranda gördüğünüz diğer pedlere değiştirerek oyun, opcode 01C2), daha sonra onlar hakkında konuşacağımız başka parametreler var. Sonuncusu, animasyonun motor tarafından ne kadar süreyle yürütüleceği, bir bekleme gibi davranacağı ve motorun o zamana kadar kodu yürütmesini durduracağı zamandır. Parametreler ve değişkenler Farklı parametre türleri vardır: * __Tamsayı__ (veya int), "1" veya "-2" veya "321" veya başka herhangi bir tam sayı olarak da adlandırılır. * __Floats__ veya kayan nokta değerleri, "1.2" veya "-2372.3845" gibi herhangi bir ondalık sayı. Kayan nokta sayıları hakkında daha fazlası. * `$PLAYER_ACTOR` global değişkeninde saklanan CJ (YOU) gibi bir __actor__ veya `$SA_RYDER`da Ryder oyun içi karakteri veya Ryder'ın görevinde ordu üssünün kapısını izleyen asker yerel değişkende saklanır "@102" Değişken, bir oyuncuyu (veya bir arabayı) tutarken (hafızada tutarken) tutamaç olarak adlandırılır, sonraki nesnenin adı, onu yakaladığınız tutamaç formu olacaktır. * __Bir "araba"__ (kodlamada her araç -uçan, süzülen, doğrayan- bir arabadır), genellikle kullandığınız araba gibi `$PLAYER_CAR`, yerel veya küresel değişkenlerde depolanan aktörler gibi. Bir değişkende bir araba stoklarken aktörlerle aynı, buna tutamaç denir. * __Model tanımlayıcısı__ `#CHEETAH`, `#ARMY`, `#PARACHUTE` gibi, bir arabanın, aktörün, nesnenin 3B modeli. gta3.img'de depolanan dff dosyalarını ifade eder. Ve arabalar, aktörler veya nesneler yaratmak için bu modelin yüklenmesi (belleğe alınması) gerekir. * __Kısa metin dizileri__ (8 bayt uzunluğunda) `'STRAP_4'' veya ''MTIME3'' gibi, motorun içinden geçerken "okuduğu" ve dosyayı - veya dosyanın - adlı satırını aradığı harfli kelimelerdir. Her zaman maksimum 7 karakterdir. Gxt, animasyon dosyaları, IPL dosyaları (iç garaj vb...) veya konu adları, metin girişleri (gxt dosyasından), animasyonlar (animasyon dosyasındaki animasyon) ve diğer birçok şey gibi dosya adlarını ifade eder. Endişelenmeyin, bu sadece parametre için bir formattır. * __Uzun metin dizileri__ (16 bayt veya daha uzun) `"ATM"` veya `"JEANSDENIM"` gibi. Yukarıdakiyle tamamen aynıdır, yalnızca parametre için bir biçimdir, ancak bu daha uzun sözcükleri işleyebilir (Sanny'nin ReadMe'si 255 karaktere kadar diyor) ve sözcük 7 karakterden kısa olsa bile kullanılır. Dizeler hakkında daha fazla açıklama bulabilirsiniz (Seemann'dan gelen bu mesajda hem kısa hem de uzun) * __Etiketler__ `MAIN_4059` gibi, yalnızca atlamalarda (ve gosub'larda) kullanılır çünkü bu işlem kodları (`0002: jump @`; `004D: jump_if_false @`; `0050: gosub @` ; veya `004F: create_thread @` ) yapar motora yolculuk (atlama) etiketi söylendi. Etiket, önünde : ile gösterilir. example:\ **CODE** ```sb3 :label 0001: wait 0 ms 0002: jump @label ``` Değişkenler, içinde bir şey sakladığınız küçük bir bellek parçasıdır. Tanımladıktan sonra motor, değiştirilene veya temizlenene kadar onu hatırlayacaktır, değişkeni kaydettikten sonra bile mevcut olmaya devam edecektir (kaydetmek tüm değişkeni kaydeder ve oyun yüklendiğinde yeniden yükler). Tıpkı bir aktör yarattığınızda, ona bir değişken atarsınız, bu onun adı gibi olur. Bunlara tamsayı, değişkenler, aktörler, araçlar, metin dizileri (özel biçim) gibi her şeyi koyabilirsiniz. O zaman iki büyük değişken ailesi vardır: * Global değişkenler: `$` ile başlayarak temsil edilirler. Globaller main.scm'de her yerde kullanılabilir. Görevin ana bölümünde bir şeye ayarlarsanız, yine de erişilebilir olacak ve bir şeye eşit olacaktır. Örnekler: `$ONMISSION`, `$PLAYER_ACTOR`. * Yerel değişkenler : Yereller, sonunda "1@", "2@" gibi bir "@" ile temsil edilir. Her iş parçacığının ('004F' tarafından oluşturulan) kendi yerel kümeleri vardır, örneğin ':thread_A' içindeki '0@', ':thread_B' içindeki '0@' ile aynı değildir. Ana kısımda `33@` üzerinden __yapamazsınız__ (bu yerel bir zamanlayıcıdır, daha fazla ayrıntı için aşağıya bakın). Ancak görevde en fazla `1024@` kullanabilirsiniz. Yerel zamanlayıcılar : "32@" ve "33@" dahili yerel zamanlayıcılardır, geçen milisaniye sayısını sayarlar. Örneğin, 0 olarak ayarlarsanız, 10 saniye bekleyin ve bakın ne kadar '32@' veya '33@' değerinde 10 000 bulacaksınız. 1 000 olarak ayarlarsanız ve bakmadan önce 5 saniye beklerseniz eşit olacaktır 6 000 ms'ye kadar. Kısa ve uzun metin dizileri : metin dizileri, özel değişken biçiminde saklanır. Metin dizisi a Sanny'de kırmızıyla vurgulanır. 4 tür metin değişkeni vardır 2 kısa (8 bayt, `'STRING'') dizeler için, 2 uzun dizeler için (16 bayt, `"LONGSTRING"`): #### Kısa dizeler için - `s$1169` veya `s$ACTOR_SPEECH_GXT_REFERENCE` gibi, diğer globaller gibi her zaman erişilebilen global bir metin dizesi. - "170@s" veya "5@s" gibi, yalnızca kendi ileti dizisinde erişilebilen yerel bir metin dizesi. #### Uzun dizeler için - genel bir metin dizesi : `v$1225` veya `v$MYLONGSTRING`, kısa dizeyle aynı, ancak uzun olanlar için... - yerel bir uzun dize : `28@v`, kısa olanlarla aynı... ANCAK dikkat edin, "example0@s" için varsa "0@" yerine "0@" koyun, bu aynı değişkendir ancak yalnızca farklı bir biçimdedirler. Ve size bir tamsayı ve bir kayan noktanın oyun motorundan farklı olduğunu hatırlatmak (veya söylemek) istiyorum ( 1, 1.0 ile aynı değildir !!!!!). İşlem kodunun bir değişkene ihtiyacı varsa ve ona bir int verirseniz (ve tersi), oyun ÇÖKECEKTİR. Bir tamsayıyı "008D" işlem koduyla kayan sayıya dönüştürebilirsiniz. "Eğer"ler ve koşullu işlem kodları "if" denen şey, bir şeylerin olup olmadığını kontrol eden bir kod yapısıdır. Beğenmek:\ if\ I die\ if not then --> I will finish my mods\ --> you give all my unfinished mods to the GTA community In Sanny Builder it would look like ```sb3 00D6: if 0118: actor $PLAYER_ACTOR dead 004D: jump_if_false @finishmods 0002: jump @givemods ``` Motor, kontrol edilen bir şey olursa doğrudan hareket edecek, eğer olmazsa başka bir etikete atlayacaktır (`004D: jump_if_false @`). İf'in (olabilecek ya da olmayacak) bir şeyine koşul denir, onlar için koşullu opcode kullanırsınız !!!!!!! Ve bir tanesine en fazla yedi koşul koyabilirsiniz, eğer !!!!!!! O zaman iki tür çoklu koşul vardır yani Eğerler: When you have multiple conditions you might want to have __all__ conditions true (happens) or __at least one of them__ true.
  • if __and__ is when you want all the conditons to be true for passing through like: ```sb3 00D6: if and 08AB: external_script 38 (BAR_STAFF) loaded 08AB: external_script 37 (CUSTOMER_PANIC) loaded 08AB: external_script 28 (BAR_AMBIENCE) loaded 08AB: external_script 35 (DABCE) loaded 004D: jump_if_false @INTMAN_1080 ``` So here you can see that if all those external scripts are loaded it goes through but then if any of them isn't it will jump to label `:INTMAN_1088`. You can think it as if this __and__ this __and__ this is OK then go throught if not jump because it's false.
  • if or is when you only need one of those conditions to be go through like: ```sb3 00D6: if or 03B0: garage 38 'BODLAWN' door_open 03B0: garage 37 'MODLAST' door_open 03B0: garage 28 'MDSSFSE' door_open 03B0: garage 35 'VECMOD' door_open 004D: jump_if_false @MAIN_4336 ``` Here it's if any of these door is open keep going but if not jump to `:MAIN_4336`. You can think it as if this or this __or__ this __or__ happens go througt but if there isn't any that is true jump because it's false.
__And__ ve __or__ mantıksal işleçler olarak adlandırılır. Tek bir koşulda bir veya veya bir ve eklemenin ne kadar anlamlı olmayacağına dikkat edin.
