Here you can set up source file names, labels format, and identifiers case.

There the file name masks are present. The upper line contains path and name of the text file which is created at decompiling. Bottom lines contain paths and names of two binary files SCM and IMG which are created at compiling. All lines have the same syntax. It uses special words meaning part of full file name:

$dir – file path
$name – file name
$ext – file extension

Example, if you decompile the file C:\MyDir\main.scm, then

$dir = C:\MyDir;
$name = main
$ext - .scm
So, if the mask is $dir\$name.txt, the output file will be named C:\MyDir\main.txt

Here you may choose one of the various name formats for the labels used at decompiling.

Global Offset: all labels will have a numeric name, for example @12345. The number means this label’s offset in the file being decompiled.

Thread+Local Offset: label name will consist of two parts: a name of the thread where the label is (the name_thread opcode is used for this), and additionally offset from this thread’s beginning, for example @MAIN_12.

Thread + Global Offset: this option is similar to the previous one, except that after the thread name there will be not a local offset but the global one (as the first case), for example @HELP_34567.

If you want the offsets to be hexadimal, select the proper option. By default, all offsets are decimal.

These options make the decompiler use or not the custom names for various script literals.

The labels names are contained in the file CustomLabels.ini in the folder SB\data\<game>. This file contains the list of labels names and offsets (for main.scm version 1.0). If the first box checked, the decompiler names the label as defined.

The variables names are present in the file CustomVariables.ini for each game. It looks like the list of global variables addresses (DMA) and their textual representation. If this box checked, the decompiler gives to the global variable its custom name.

The arrays names list are present in the file CustomArrays.ini. Its syntax is as follows: the first number is the global variable name (DMA) which is the first element of the array (array name), the second – array size, the third – custom name. This option is used in order to decompiler adds an index to some global variables being an array. So, these variables become indexed ones.

This option has two meanings. The decompiler names global variables, labels, arrays using the selected letters case. The compiler writes the strings in the SCM using the selected letter case also.
