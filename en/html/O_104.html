You may set various syntax highlighting schemes on this page. Each code element has 3 properties: Foreground, Background, and Style.
3 style elements are available: B – bold, I – italicize, U – underlined.

The selected highlighting scheme is also used at saving the code source in the RTF or HTML format.

Code elements:

Comments – the text after double-slash // or between braces {}.

Labels – the identifiers beginning with an @.

Variables – the identifiers having a dollar-sign $ in the beginning (global variables), or ending with an @ (local variables). Also the aDMA type (begins with an &) and the arrays are highlighted same as variables.

Keywords – the special words stored in the file keywords.ini (one for all games) and keywords.txt (separate file for each game).

Numbers – the integer and floating-point numbers

Strings – the text between “…” and ‘…’.

Models – the identifiers beginning with a #.

Classes – the part of the class command: the identifier before the point.

Commands – the part of the class commands: identifier after the point.

Directives – the comment containing a $ immediately after the opening {.
