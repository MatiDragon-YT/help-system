This page contains the (de)compiler options mostly.



Sanny Builder needs certain game files while working, so you have to choose a correct directory where the game is installed. You may input the path manually or by selecting the directory via the dialog appearing when you press the button.


The program also needs the INI file containing the opcodes list. You may set the path to this file here (manually or by selecting file dialog appearing when you press the button).



There is a list of available translations for Sanny Builder. The translation files are present in the folder \lang. Choose desired language for the program interface in this list. The translation instructions are contained in the file "How to translate.txt" in the folder \lang.



You may associate different file extensions with Sanny Builder. When a checkbox is selected, your Windows Registry is changed and you may open the files with this extension simply by clicking it. To cancel the association, unselect a proper checkbox.



Here you see common options of the (de)compiler.

Always overwrite output file

If this one checked, the decompiler writes decompiled data in the output file disregarding if it exists or not. If not, and the text file with such name already exists, the decompiler won’t rewrite it but create a new text file with index (for example, main[0].txt).

Show progress

While program executes (de)compiling process and if this box selected, there is a progress line in the status bar. Otherwise status bar have words: Please wait. Showing progress slightly slows the execution time.

Show warning

This option has the meaning only when San Andreas scripts are compiling. If the game is running during the compiling process, the file script.img containing the part of the scripts could not be rewrote as the game uses this file. The compiler shows a warning about it. You may disable this warning by checking this box.

Manual IMG opening

When the decompiling process started, the program searches for the file script.img containing some game scripts. If this file is not found in the same folder with the SCM file or in the San Andreas\data\scripts\ folder, the error message is shown. If this option toggled, you can select needed img file manually.

Quick game loading

Sanny Builder is able to skip SA startup splash screens (F8). If this ability causes an error in your game version, disable this option.

Check conditions

This option has two meanings. First, the program replaces the number of conditions in the IF opcode with AND or OR. Second, the program is able to compile the IF..END construct, and calculate needed IF parameter in the conditions with words AND or OR.

Ranges check

The number of accessible local variables is limited. When this option is checked, the compiler checks the correct using of them.

Write opcodes

This option turns the decompiling output from high-level to low-level and vice versa. If this option is unchecked, the decompiler uses available classes and keywords instead of opcodes. In addition, there are no opcodes for simple math expressions. If option checked, all opcodes are present in the output file.

Replace mission numbers

When this option checked, the decompiler replaces mission number in the opcode start_mission with mission name. The mission name is the label name defined in the file header.
This name also could be used for quick movement to the mission code.

Insert original mission names

When this option checked, the decompiler adds predefined original mission names as comments for the opcode start_mission and for the line DEFINE MISSION in the file header.

Add extra info to SCM

If this option is checked the compiler adds extra information to the end of the file main.scm. This info will be used later when this file is decompiled in order to bring it to maximum conformity with original source. The following data is added to the end of the main.scm:  hex..end constructs offsets, global variables names and so on. Pay attention that the file compiled in such way cannot be decompiled in older SB versions (except the case when ignore_unknown option is enabled).
