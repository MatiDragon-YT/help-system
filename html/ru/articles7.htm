Quick Start. Часть 1
Начнем с того, что, если поначалу вам непривычны новшества в Sanny, то просто включите опцию Write opcodes в настройках и используйте его также как SaMB (а поковыряв прочие настройки, можно вообще сделать два редактора практически одинаковыми внешне). При этом воз и тележка разнообразных фич и удобств проги, а также высокая скорость работы по-прежнему будут в вашем распоряжении.

Вы также откроете для себя очень много полезного, внимательно прочитав readme.txt из комплекта Sanny Builder. Там указаны абсолютно все отличия и новшества Sanny по сравнению с редактором Mission Builder. К сожалению в настоящий момент в ридми практически не рассматриваются основы скриптинга (создание потоков, миссий и т.п.). Однако эти основы являются общими для обеих программ и, если у вас возникают проблемы, вы можете ознакомиться с этим сборником вопросов по Mission Builder'у. Большая часть того, что там написано, действительно и для SB.
SB может использоваться для редактирования скриптов всей линейки GTA3 (GTA3, VC, SA).

Краткий перечень важных отличий между программами:

а) В переходах в SB используется символ @ вместо нечитаемого на многих форумах символа английского фунта (JJ): jump @Label, gosub @Label
Важный момент, что метки, независимо от их положения в коде, записываются одинаково. В SAMB вам приходилось бы писать в миссиях метки с одним знаком J, а в потоках с двумя - JJ.
Локальные переменные, следовательно, записываются с номером переменной в начале (а не в конце как в SAMB). Например:

@1, @2, @3 - формат SAMB
1@, 2@, 3@ - формат SB

б) Множество простых скриптовых команд можно записывать БЕЗ опкодов, таких как wait, end_thread, jump и т.п. Данные слова именуются кейвордами и заменяют собой часто используемые опкоды. Например:

wait 0 - аналог 0001: wait 0 ms
if - аналог 00D6: if 0
if and - аналог 00D6: if 1-7
if or - аналог 00D6: if 21-27
jf @Label - аналог 004D: jump_if_false @Label

и многие другие. Список возможных кейвордов можно вызвать в редакторе нажатием Ctrl+Space.

в) Работа с числами. Все арифметические действия также можно (и нужно) записывать без опкода, например вот так:

$Var = 1
$Var = 45.5
$Var += 5
$Var *= 0.01

Данная возможность полностью перекладывает на компилятор работу по подбору нужного опкода для выражения. Вам лишь нужно указать знаки действий (+, -, >, ==, и т.д.), причем опкоды подбираются как для локальных, так и для глобальных переменных. Sanny Builder поддерживает ВСЕ виды математических знаков, включая <=, <>, <. Последние удобно использовать в негативных условиях (со словом not). Например вместо:

if
not $Var == 1
jf @label

можно написать

if
$Var <> 1
jf @label

В том случае, если в выражении оба параметра - переменные, компилятор не может сразу определить, какого они типа - целые или дробные числа. В этом случае используйте конструкцию VAR..END, т.е вот так:

var
1@: Int
//сообщаем компилятору, что переменная 1@ будет содержать целые числа(integer)
$Var2: Float
//а переменная $Var2 будет содержать дробные числа(floating-point)
end

$Var2 = 45.5
if
$Var = 1@
jf @Label2

Кроме того, существуют такие удобные функции, вроде Inc (аналог $Var = $Var + 1), Random (генерирует случайные числа, заменяет сразу два опкода), и другие, о них подробнее написано в ридми.

г) Комментарии. В SB можно закомментировать строку символом двойного слеша //,
например:

01F0: set_max_wanted_level_to 6
// все что здесь и до конца строки игнорируется компилятором

а также блоками { }, которыми можно закомментировать сразу несколько строк или часть строки.

01F0: set_max_wanted_level_to {эта часть игнорируется} 6

д) Можно не заморачиваться и не писать вообще список DEFINE OBJECTS, поскольку компилятор подсчитывает используемые объекты самостоятельно.

Возможности самого редактора (IDE) можно посмотреть здесь.

Серьезно поэкспериментировав с Sanny, вы поймете, что работать с его многочисленными фишками реально проще, поскольку они избавляют вас от множества рутинной работы.
