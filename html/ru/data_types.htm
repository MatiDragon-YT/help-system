Типы данных

Код разделен на отдельные элементы, каждый из которых обозначается по-своему и выполняет конкретную функцию в скрипте.

$ - обозначение глобальных переменных. Их значения могут использоваться в любом месте кода.

Пример:

0004: $MyVar = 100

@ - в начале: метка

Пример:

0002: jump @MyLabel

@ - в конце: локальная переменная. Их значения используются только в пределах одного потока

Пример:

0006: 100@ = 10

'...' - текст (макс. 15 символов).

Допускается пустое значение: '  '

Пример:

03A4: name_thread 'MAIN'

"..." - текст (максимум определяется опкодом)

Также вы можете использовать в тексте одинарные кавычки или двойные кавычки.
Для двойных кавычек необходимо использовать специальный префикс \.

Допускается пустое значение: "  "

Пример:

0662: write_debug_message "Hello, \"world\"! 'Here we go!'"

s$ - глобальная стринговая переменная

Пример:

05A9: s$MyString = 'GLOBAL'

@s - локальная стринговая переменная

Пример:

05AA: 1@s = 'LOCAL'

v$ - глобальная стринговая переменная

Пример:

06D1: v$MyString = "LONG_GLOBAL"

@v - локальная стринговая переменная

Пример:

06D2: 1@v = "LONG_LOCAL"

# - имя модели из одного из IDE - файлов игры

Пример:

0247: request_model #CELLPHONE

& - прямой байтовый адрес (ADMA). Читает и пишет значения по указанному адресу Может использоваться в тех же случаях, когда используется тип $.

Пример:

&57 += &120(&231,4i)

(примечание: число после & - локальное смещение внутри файла main.scm, относительно его начала; адрес, куда будет записано значение)


0x - число в шестнадцатиричном формате.
-0x - отрицательное значение

Пример:

0004: $var = -0xBB08

Число должно быть в интервале -80000000..7FFFFFFF
